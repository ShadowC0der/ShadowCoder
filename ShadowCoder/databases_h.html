Databases (Higher Level)

📌 What is a Database?

A database is an organised collection of related data stored electronically so it can be easily accessed, managed, and updated.

📊 Tables, Records & Fields

  • Table: Collection of related data
  • Record: One row in a table (one item/person)
  • Field: One piece of data (e.g. name, age)

🔑 Keys in Databases

Primary Key

A field that uniquely identifies each record in a table.

Foreign Key

A field that links one table to another by referencing a primary key.

🔗 Relationships

  • One-to-One: One record links to one record
  • One-to-Many: One record links to many records
  • Many-to-Many: Multiple records link to multiple records

💡 Example

Students table and Subjects table:

  • StudentID (Primary Key)
  • SubjectID (Primary Key)
  • StudentID in Subjects = Foreign Key

📐 Normalisation (HL)

Normalisation is the process of organising data to reduce duplication and improve efficiency.

  • Removes duplicate data
  • Reduces errors
  • Improves structure

🧪 Exam Questions

Q1: Define a primary key. (3 marks)

Answer: A field that uniquely identifies each record in a table.

Q2: Explain the difference between a primary key and a foreign key. (6 marks)

Answer: A primary key uniquely identifies records in a table, while a foreign key is used to link two tables together by referencing a primary key.

🎯 Exam Tips

  • Always say “uniquely identifies” for primary key
  • Say “links tables together” for foreign key
  • Use clear table examples for higher marks
  • Structure answers: definition → explanation → example
